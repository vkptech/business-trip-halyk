Halyk SuperApp — технологическая архитектура
Участники от HalykBank: Олег Князев
Go / Golang .NET CORE Claude AI Kafka
  • Приложение не использует Istio / Envoy Gateway в качестве service mesh; применяется канареечный деплой для постепенного выкатывания обновлений.
  • Основной язык бэкенда — Go (Golang); коллеги подтверждают курс на Go как стратегический выбор для высоконагруженных сервисов.
  • Для обмена сообщениями используется Kafka; в ходе обсуждения затронуто сравнение Kafka vs RabbitMQ применительно к специфике банковских сервисов.
  • Аутентификация реализована на собственном Auth Service без зависимости от сторонних SSO-провайдеров по умолчанию.
  • Фронтенд — AngularJS; коллеги пошли путем построения webview и микрофронтендов.
  • В качестве использования AI инструментов они используют GitHub Copilot. Начинают работы по использованию Claude Code
  • В Kafka применяются идемпотентные ключи для обеспечения семантики exactly-once и предотвращения дублирования сообщений.

CREATIO — оценка Low-code BPM-платформы
Участники от HalykBank: Нұрсұлтан Темірбекүлы, Нурлан Терликбаев
Low-code / BPM требует анализа
  • Платформа произвела положительное первое впечатление; требует детального анализа применительно к задачам Тенге Банка.
  • Необходимо дополнительно оценить соответствие платформы текущим и перспективным бизнес-процессам, а также совместимость с существующим технологическим стеком.
  • Коллеги из Halyk Bank эксплуатируют устаревшую версию; для демонстрации актуальной версии продукта запланирована онлайн-встреча с вендором.
Следующий шаг: организовать Zoom-демо с вендором CREATIO для оценки новой версии платформы.
3
Online Bank (ДБО ЮЛ) — технологическая архитектура
Участники от HalykBank: Жалгасым Сериков, Александр Князев, Шыңғыс Хамзин
Kotlin / Java Spring Kafka PostgreSQL Claude Code
  • Технологический стек: Kotlin + Spring Framework; акцент на типобезопасность и лаконичность кода.
  • СУБД: Oracle; активно прорабатывается стратегия миграции на PostgreSQL, в том числе с применением Yandex SPQR — прокси-слоя для горизонтального масштабирования PostgreSQL.
  • Для AI-ассистирования разработчиков используются GitHub Copilot и Cursor; рассматривается приобретение пилотной лицензии Claude Code — что подтверждает актуальность нашей собственной потребности в данном инструменте.

Halyk Fin Service — архитектура платёжного сервиса
Участники от HalykBank: Иван Толмачев
Golang / Java PostgreSQL E-com / Payments
  • Halyk Fin Service — дочерняя компания внутри Halyk Group, специализирующаяся на обработке e-commerce транзакций и платёжных операций; фактически является платёжным шлюзом группы для онлайн-коммерции.
  • Компания работает с высоконагруженными платёжными потоками, что предъявляет повышенные требования к надёжности, идемпотентности операций и скорости обработки транзакций.
  • Технологический стек ориентирован на Go и Java; для хранения данных применяется PostgreSQL — согласуется с нашим собственным вектором развития.
  • Опыт команды в части обеспечения согласованности данных при платёжных операциях, обработки ошибок и организации retry-логики оказался практически применим для наших задач в рамках ДБО ЮЛ и QR-платежей.
  • Взаимодействие подтвердило перспективность более тесной технической кооперации с Halyk Fin Service в части стандартизации платёжных интеграций внутри группы.

ДАБП — практика применения Camunda BPM
Участники от HalykBank: Саят Шыңғыс, Алмас Амрин
Camunda External Tasks
  • Команда ДАБП активно применяет HTTP Connectors и паттерн External Tasks для интеграции внешних систем с Camunda — опыт непосредственно применим к нашему проекту BPM MVP2.
  • Параметр Lock Duration для External Tasks настраивается индивидуально под каждый тип задачи; коллеги поделились рекомендациями по оптимальным значениям.
  • Коллеги дали рекомендации по referenceId — это обеспечивает трассируемость и корректную корреляцию сообщений.
  • Получен ряд ценных практических рекомендаций по архитектуре Camunda-процессов, которые будут задокументированы и переданы команде BPM.
  • Рассматривают переход на Camunda 8 версию. Она дорогая по лицензии, также нужно будет переписывать текущие бпм процессы на Camunda 8. Это долго и дорого, ищут альтернативы

AI в Halyk SuperApp — прикладные сценарии
Участники от HalykBank: Олег Князев и команда Halyk SuperApp
Claude AI Agent AI-assisted Dev
  • Claude AI развёрнут в роли AI-агента бизнес-аналитика: автоматизирует сбор и формализацию требований, снижает нагрузку на аналитиков при первичной обработке запросов.
  • Интеграция AI в процессы разработки (Copilot, Cursor, Claude Code) рассматривается как инструмент повышения скорости и качества, а не замены специалистов.
  • Опыт Halyk SuperApp подтверждает технологическую зрелость AI-инструментов для применения в банковской среде.
